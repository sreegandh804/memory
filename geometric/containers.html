Create esm image measurement container

Parameters

  • img_mean – Camera-relative co-ordinates and image features [batch_size, timesteps, height, width, 3 + feat]

  • cam_rel_mat (array, optional) – The pose of the camera relative to the current agent pose. Default is identity matrix [batch_size, timesteps, 3, 4]

  • img_var – Image depth and feature variance values, assumed all zero if None. [batch_size, timesteps, height, width, 1 + feat]

  • validity_mask (array, optional) – Validity mask, for which pixels should be considered. Assumed all valid if None [batch_size, timesteps, height, width, 1]

  • pose_mean (array, optional) – The pose of the camera relative to the current agent pose, in rotation vector pose form. Inferred from cam_rel_mat if None. [batch_size, timesteps, 6]

  • pose_cov (array, optional) – The convariance of the camera relative pose, in rotation vector form. Assumed all zero if None. [batch_size, timesteps, 6, 6]

  • dev_str (str) – Device string to use, default is to use img_mean.
